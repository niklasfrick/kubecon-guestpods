# Phase 3 — UI Design Contract

> Visual and interaction contract for the admin panel. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (plain CSS with custom properties) |
| Icon library | none (inline SVG / Unicode characters) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

The project uses plain CSS with `:root` custom properties defined in `web/src/style.css`. No Tailwind, no shadcn, no component library. The admin panel MUST extend the existing `style.css` token system — do not introduce a second styling approach.

Monospace font for K8s-style identifiers: `ui-monospace, "SF Mono", Menlo, Consolas, monospace` (already defined as `--font-mono`).

## Layout Contract

### Overall Structure
Single-column, full-viewport layout. No sidebar, no tabs. Three vertically stacked zones visible without scrolling on a 1080p display:

```
+--------------------------------------+
|  [heading]          [status indicator]|  Zone 1: Header (fixed height)
+--------------------------------------+
|                                       |
|  [ CLOSE SUBMISSIONS ]  (big toggle) |  Zone 2: Control + Stats
|                                       |
|  Total Pods: 247  Namespaces: 12     |
|  Top: DE (89) US (45) GB (31)        |
+--------------------------------------+
|  submission list (scrollable)         |  Zone 3: Pod List (flex-grow)
|  - name / ns / emoji / timestamp [x] |
|  - name / ns / emoji / timestamp [x] |
|  - ...                                |
+--------------------------------------+
```

### Zone 1: Header
- Height: 48px
- Horizontal flex: heading left, status indicator right
- Heading text: "Cluster Admin" in 20px semibold
- Status indicator: green dot + "Accepting pods" (or red dot + "Rejecting pods")

### Zone 2: Control + Stats
- Padding: `--space-lg` (24px) vertical
- Toggle button: full-width, 56px height, 16px semibold text, `border-radius: 8px`
  - OPEN state: `--color-accent` background, `--color-dominant` text, label "Close Submissions"
  - CLOSED state: `--color-destructive` background, `--color-text-primary` text, label "Reopen Submissions"
- Stats row: horizontal flex below toggle, gap `--space-lg` (24px)
- Each stat: value in 28px semibold above label in 14px regular secondary text
- Top locations: inline list "DE (89) US (45) GB (31)" in 14px mono

### Zone 3: Pod List
- Flex-grow, overflow-y auto (scrollable)
- Section heading: "Live Pod Feed" in 14px semibold secondary, with total count badge
- Each row: horizontal flex, height 48px, padding `--space-sm` horizontal
  - Emoji (28px), name (16px semibold, truncated with ellipsis at 200px max-width), namespace in mono (14px secondary), timestamp relative (14px secondary), delete button (right-aligned)
- Row dividers: 1px `--color-border` bottom border
- New rows animate in via `slideDown` keyframe (200ms ease-out, already exists in `style.css`)
- Delete button: 32x32px, text "x" or inline SVG trash icon, `--color-text-secondary` default, `--color-destructive` on hover

### Login Form
- Centered card layout (reuse existing `.card` class, max-width 400px)
- Heading: "Cluster Admin" in 20px semibold
- Subheading: "Enter admin password to continue" in 14px regular secondary
- Password input: reuse existing `.text-input` class, type="password"
- Submit button: reuse existing `.submit-btn` class, label "Authenticate"
- Error state: reuse existing `.error-banner` for wrong password

## Interaction Contract

### Toggle Button
- Single click toggles open/closed state
- Optimistic UI: button changes state immediately, reverts on API error
- Disabled for 500ms after toggle to prevent double-tap under stage nerves
- API: `POST /api/admin/toggle`

### Delete Pod
- Click delete button on row -> native `window.confirm("Delete this pod?")` dialog
- On confirm: row fades out (opacity 0 over 150ms), then removed from DOM
- API: `DELETE /api/admin/submissions/{id}`
- Optimistic removal: remove from list immediately, re-add on API error

### Login
- Password field auto-focused on mount
- Enter key submits (standard form behavior)
- On wrong password: error banner "Invalid password. Check with the cluster operator." for 3 seconds
- On success: replace login form with admin dashboard (no page reload)

### Real-time Updates
- SSE connection to `/api/submissions/stream` (same endpoint as viz)
- On `submission` event: prepend new row to pod list, increment stats
- On `deletion` event: remove row from pod list, decrement stats
- Stats recomputed client-side from the pod list state (no separate stats endpoint polling)

### Auth Check on Load
- On mount: `GET /api/admin/status` to check session validity
- If valid session: render dashboard
- If no/invalid session: render login form
- Loading state: brief "Checking session..." text while auth check resolves

## Accessibility

- Toggle button: `aria-pressed` attribute reflects open/closed state
- Delete buttons: `aria-label="Delete pod {name}"` for screen readers
- Status indicator: `aria-live="polite"` region so screen readers announce state changes
- Login password field: `autocomplete="current-password"` for password manager support
- Focus management: after login success, focus moves to toggle button
- All interactive elements meet 44px minimum touch target
- `prefers-reduced-motion`: existing media query in `style.css` disables animations globally
