# Phase 2 — UI Design Contract

> Visual and interaction contract for the real-time K8s cluster visualization. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Preact + imperative Canvas 2D) |
| Icon library | none (emoji-only, no icon sprites) |
| Font | system-ui stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` (matches Phase 1 `--font-family`) |

**Note:** This phase is primarily Canvas-rendered. The design contract specifies Canvas drawing parameters (ctx.font, ctx.fillStyle, ctx.shadowBlur) rather than CSS classes. Only two DOM overlays exist (StatsOverlay, HoverCard) and those inherit Phase 1's CSS custom properties from `style.css`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Padding inside namespace label badges (vertical) |
| sm | 8px | Gap between stat items in overlay, namespace label badge horizontal padding |
| compact | 12px | HoverCard inner padding, stats overlay vertical/horizontal inner padding |
| md | 16px | HoverCard outer margin from viewport, overlay margin from viewport edge |
| lg | 24px | Canvas hull padding around cluster nodes |
| xl | 32px | Not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Canvas hull padding uses 25px (nearest practical value for convex hull expansion; not a DOM spacing token).
- Namespace badge gap above hull uses 8px (`sm` token) for vertical clearance between label and hull top edge.

## Color

**Color distribution: ~60% dark background (#0f172a slate-900 canvas fill), ~30% neutral surfaces (translucent overlays, namespace badges, cluster boundary fills), ~10% level-color pod accents (5 homelab level colors).**

### Canvas Palette

| Role | Value | Usage |
|------|-------|-------|
| Background (dominant 60%) | `#0f172a` (slate-900) | Full-screen Canvas fill, edge-to-edge |
| Cluster boundary fill | `rgba(148, 163, 184, 0.12)` (slate-400 at 12%) | Translucent convex hull fill for namespace clusters |
| Namespace badge background | `rgba(15, 23, 42, 0.85)` (slate-900 at 85%) | Dark pill behind namespace label text |
| Namespace badge text | `#ffffff` | White text on dark badge |
| Pod label text | `#ffffff` | White text on colored pod body |

Source: RESEARCH.md code example `LEVEL_COLORS`, discretion area.

### DOM Overlay Colors (inherited from Phase 1 CSS custom properties)

| Role | Value | Usage |
|------|-------|-------|
| Overlay background | `rgba(15, 23, 42, 0.90)` | Stats overlay and HoverCard background (slightly more opaque than cluster fill for readability) |

## Entrance Animation Specification

| Property | Value |
|----------|-------|
| Type | Scale up from 0 + color glow |
| Easing | `easeBackOut` (d3-ease) -- overshoot and settle |
| Duration | 600ms scale-up to full size |
| Glow duration | 2000ms fade from full to 0 opacity |
| Glow implementation | `ctx.shadowColor` = level color, `ctx.shadowBlur` = `15 * glowOpacity` |
| Stagger (SSE arrivals) | 200ms between consecutive pods |
| Stagger (initial load) | 50ms between consecutive pods |
| Queue threshold | If >20 pods pending, queue excess; process 20 at a time |
| Cluster reaction | Hull boundary smoothly expands to include new pod; namespace label briefly brightens (opacity pulse 0.85 to 1.0 to 0.85 over 400ms) |

## Hover Interaction Specification

| Property | Value |
|----------|-------|
| Hit detection | `simulation.find(mouseX, mouseY, 20)` -- 20px search radius |
| Cursor | `pointer` when over a pod, `default` otherwise |
| Trigger | mousemove on canvas element |
| Debounce | none (simulation.find is O(log n) via quadtree) |
| HoverCard position | 12px to the right of the hovered pod; flip to left if within 200px of right viewport edge |
| HoverCard max width | 220px |
| HoverCard show delay | 0ms (instant) |
| HoverCard hide delay | 100ms (prevents flicker when moving between adjacent pods) |

## Screen Layout

| Property | Value |
|----------|-------|
| Canvas size | `window.innerWidth` x `window.innerHeight` (full viewport) |
| Canvas CSS | `display: block; position: fixed; top: 0; left: 0; width: 100vw; height: 100vh;` |
| DPI scaling | `canvas.width = clientWidth * devicePixelRatio; canvas.height = clientHeight * devicePixelRatio; ctx.scale(dpr, dpr)` |
| Background | `#0f172a` filled on every frame |
| Draw order | 1. Background fill, 2. Cluster boundaries, 3. Pods (back-to-front by animProgress), 4. Glow effects |
| Header/sidebar | none -- full edge-to-edge canvas |
| Background pattern | none -- clean solid dark |
| Target resolution | 1920x1080 (projector), responsive to any viewport |
| Resize handling | Debounced (200ms) resize listener; recalculate canvas dimensions and DPI; update forceCenter to new center; gentle simulation reheat `alpha(0.3)` |

## Initial Load Behavior

| Step | Description |
|------|-------------|
| 1 | Fetch `GET /api/submissions` to get all existing pods |
| 2 | Create all PodNode objects with `animProgress: 0` (invisible) |
| 3 | Pre-compute layout: `simulation.nodes(allNodes); simulation.alpha(1); for 120 ticks: simulation.tick()` |
| 4 | Start visual cascade: animate pods in at 50ms stagger ("booting up the cluster" feel) |
| 5 | After cascade completes (nodes.length * 50ms + 500ms buffer), switch stagger to 200ms |
| 6 | Connect to SSE endpoint `GET /api/submissions/stream` |
| 7 | New SSE arrivals enter the animation queue at 200ms stagger |

## Accessibility Notes

| Concern | Approach |
|---------|----------|
| Reduced motion | Respect `prefers-reduced-motion`: skip entrance animation (pods appear instantly at full size, no glow), skip cluster boundary expansion animation |
| Keyboard navigation | Not applicable -- this is a presenter-facing projector display, not an interactive form |
| Screen reader | Canvas is inherently non-accessible. Add `role="img"` and `aria-label="Live visualization of {N} attendee pods in {N} namespace clusters"` to canvas element. Update aria-label on count changes. |
| Color contrast | Pod label text (#ffffff) on all 5 level colors passes WCAG AA for large text. Namespace badge text (#ffffff) on dark badge (#0f172a at 85%) passes AAA. |
| Focus management | Not applicable -- no focusable elements on the visualization page (presenter uses mouse hover only) |
