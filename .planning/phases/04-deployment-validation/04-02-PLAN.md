<objective>
Create a Helm chart for deploying the guestbook application to a Kubernetes cluster with Cilium Gateway API ingress, Rook Ceph persistent storage, and cert-manager TLS certificates.

Purpose: The Helm chart is the deployment package for the homelab Kubernetes cluster. It must be generic with sensible defaults so it works standalone with `helm install`, while homelab-specific values (storage class, Gateway name, domain) are overridden in the separate ArgoCD gitops repo. The PVC ensures SQLite data survives pod restarts (INFR-02). The Certificate resource enables cert-manager to issue origin TLS certificates between Cloudflare and the cluster.

Output: Complete Helm chart in chart/ directory with Deployment, Service, PVC, HTTPRoute, Certificate, ConfigMap, and helpers
</objective>

**chart/values.yaml:**
```yaml
# Image configuration
image:
  repository: ghcr.io/niklas/kubecon-guestbook
  tag: ""  # Overridden by appVersion if empty
  pullPolicy: IfNotPresent

# Replica count -- MUST be 1 (SQLite single-writer constraint)
replicaCount: 1

# Application configuration
config:
  # Base URL for QR code generation (override with actual domain)
  baseURL: "https://guestbook.example.com"
  # Listen address (container-internal)
  addr: ":8080"

# Admin password -- reference an existing Kubernetes Secret
adminPassword:
  existingSecret: "guestbook-admin"
  key: "password"

# Persistence for SQLite database
persistence:
  enabled: true
  storageClass: ""  # Use cluster default; override with e.g. "rook-ceph-block" in gitops
  size: 1Gi
  accessMode: ReadWriteOnce
  # Prevent data loss on helm uninstall
  annotations:
    helm.sh/resource-policy: keep

# Service configuration
service:
  type: ClusterIP
  port: 8080

# Gateway API HTTPRoute
httpRoute:
  enabled: true
  gateway:
    name: main-gateway
    namespace: default
  domain: guestbook.example.com

# TLS via cert-manager Certificate resource
tls:
  enabled: true
  issuer: letsencrypt-prod
  issuerKind: ClusterIssuer  # ClusterIssuer or Issuer
  secretName: guestbook-tls

# Resource limits
resources:
  requests:
    cpu: 50m
    memory: 64Mi
  limits:
    cpu: 500m
    memory: 256Mi

# Health probes
probes:
  liveness:
    path: /api/health
    initialDelaySeconds: 5
    periodSeconds: 15
  readiness:
    path: /api/health
    initialDelaySeconds: 3
    periodSeconds: 10
```

**chart/templates/deployment.yaml:**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ include "guestbook.fullname" . }}
  labels:
    {{- include "guestbook.labels" . | nindent 4 }}
spec:
  replicas: {{ .Values.replicaCount }}
  strategy:
    type: Recreate  # SQLite single-writer -- no rolling update
  selector:
    matchLabels:
      {{- include "guestbook.selectorLabels" . | nindent 6 }}
  template:
    metadata:
      labels:
        {{- include "guestbook.selectorLabels" . | nindent 8 }}
    spec:
      containers:
        - name: guestbook
          image: {{ include "guestbook.image" . }}
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          ports:
            - name: http
              containerPort: 8080
              protocol: TCP
          env:
            - name: ADMIN_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: {{ .Values.adminPassword.existingSecret }}
                  key: {{ .Values.adminPassword.key }}
            - name: DB_PATH
              value: /data/guestbook.db
          envFrom:
            - configMapRef:
                name: {{ include "guestbook.fullname" . }}
          livenessProbe:
            httpGet:
              path: {{ .Values.probes.liveness.path }}
              port: http
            initialDelaySeconds: {{ .Values.probes.liveness.initialDelaySeconds }}
            periodSeconds: {{ .Values.probes.liveness.periodSeconds }}
          readinessProbe:
            httpGet:
              path: {{ .Values.probes.readiness.path }}
              port: http
            initialDelaySeconds: {{ .Values.probes.readiness.initialDelaySeconds }}
            periodSeconds: {{ .Values.probes.readiness.periodSeconds }}
          resources:
            {{- toYaml .Values.resources | nindent 12 }}
          volumeMounts:
            {{- if .Values.persistence.enabled }}
            - name: data
              mountPath: /data
            {{- end }}
      terminationGracePeriodSeconds: 15
      {{- if .Values.persistence.enabled }}
      volumes:
        - name: data
          persistentVolumeClaim:
            claimName: {{ include "guestbook.fullname" . }}-data
      {{- end }}
```

```yaml
    {{- include "guestbook.labels" . | nindent 4 }}
```

```yaml
    - {{ .Values.httpRoute.domain | quote }}
{{- end }}
```

This Certificate resource tells cert-manager to issue a TLS certificate for the guestbook domain using the configured ClusterIssuer (default: letsencrypt-prod). The resulting Secret (guestbook-tls) is referenced by the Gateway's TLS listener in the gitops repo. Conditional on `tls.enabled: true` so it can be disabled for local development.
