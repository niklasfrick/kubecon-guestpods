# Phase 1 — UI Design Contract

> Visual and interaction contract for the mobile submission form and post-submission confirmation screen. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Preact components) |
| Icon library | none (emojis are the visual language) |
| Font | system font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

Rationale: Preact (not React) rules out shadcn. System font stack eliminates font loading latency, critical for the 2-second load target on conference WiFi (INFR-04). Emojis serve as the icon system (homelab scale, country flags, confirmation).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding within homelab scale buttons, icon-to-text gaps |
| sm | 8px | Gap between homelab emoji buttons, field label margin-bottom |
| md | 16px | Form field vertical spacing, card internal padding (horizontal) |
| lg | 24px | Card internal padding (vertical), section spacing |
| xl | 32px | Page horizontal padding on mobile |
| 2xl | 48px | Top padding from viewport edge to form card |
| 3xl | 64px | Not used in this phase |

Exceptions: Homelab scale buttons have a minimum touch target of 44x44px (WCAG 2.5.5 Target Size), which is not a multiple of 4. This overrides any spacing-derived sizing for those interactive elements.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form field values, confirmation details, homelab descriptions |
| Label | 14px | 600 (semibold) | 1.4 | Form field labels ("Your name", "Where are you from?", "Homelab level") |
| Heading | 24px | 600 (semibold) | 1.2 | Confirmation heading ("Pod deployed!"), form title if needed |
| CTA | 16px | 600 (semibold) | 1.0 | Submit button text ("kubectl apply") |

Notes:
- Body text at 16px prevents iOS Safari zoom-on-focus for input fields (iOS zooms if input font is below 16px).
- Only 2 weights used: 400 and 600. No bold (700), no thin (300).
- Homelab emoji buttons render at 28px font-size to ensure legibility on mobile without being oversized.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F172A` (slate-900) | Page background — dark "terminal" theme |
| Secondary (30%) | `#1E293B` (slate-800) | Form card background, input field backgrounds |
| Accent (10%) | `#38BDF8` (sky-400) | Submit button background, selected homelab scale button ring, "Status: Running" indicator |
| Destructive | `#EF4444` (red-500) | Validation error text, profanity rejection message |

Additional tokens:

| Token | Value | Usage |
|-------|-------|-------|
| Text primary | `#F8FAFC` (slate-50) | Headings, body text, input values |
| Text secondary | `#94A3B8` (slate-400) | Labels, placeholders, helper text, homelab descriptions |
| Text success | `#4ADE80` (green-400) | "Pod deployed!" heading, "Status: Running" dot, checkmark |
| Border | `#334155` (slate-700) | Input field borders, card border, homelab scale track border |
| Border focus | `#38BDF8` (sky-400) | Input focus ring (matches accent) |

Accent reserved for: submit button background, selected homelab scale indicator, input focus rings, and the "Status: Running" colored dot on the confirmation screen. Never used for decorative elements, backgrounds, or non-interactive text.

Dark theme rationale: A terminal/K8s dashboard aesthetic reinforces the "kubectl apply" and "Pod deployed!" theming. Dark backgrounds also work well on projector screens and conference lighting.

```
|  |    [emoji] [emoji]         |  |
```

- Card: `max-width: 400px`, centered horizontally with `margin: 0 auto`

- All inputs: `height: 48px`, `border-radius: 8px`, `padding: 0 12px`
- Submit button: full-width within card, `height: 48px`, `border-radius: 8px`
- Homelab scale: 5 buttons arranged in a single row, each 44x44px minimum, `gap: 8px`, centered

```
|  |     "Pod deployed!"        |  |

|  |   [emoji]  [name]          |  |
|  |   ns/[country_code]        |  |

```

- Same card container as form (consistent visual)
- All content center-aligned
- Checkmark: Unicode character or CSS-drawn, colored with success token
- Namespace line uses `font-family: monospace` for K8s authenticity
- "Look up at the screen" text uses text-secondary color, 14px italic

### Responsive behavior

- Below 375px: card padding reduces from 16px horizontal to 12px. Homelab scale wraps to 3+2 layout if needed.
- Above 480px: no changes — this is a mobile-only form, no desktop layout needed.
- No media queries beyond the 375px breakpoint. The form should work in a mobile viewport and is not designed for desktop use.

## Interaction Contracts

### Form Submission Flow

1. **Initial state:** Form renders with all fields empty. Submit button shows "kubectl apply" in disabled state (accent background at 50% opacity). Name input is auto-focused.
2. **Validation gate:** Submit button becomes enabled (full accent opacity) only when all three fields have values: name is non-empty, country is selected, homelab level is selected.
3. **Submitting state:** On tap, button text changes to "Deploying..." with a small CSS spinner. All inputs become disabled. Button remains full-width, same height.
4. **Success state:** Entire form card smoothly transitions (200ms CSS opacity fade) to the confirmation screen. No page navigation — same DOM node, state swap.
5. **Error state:** An error banner appears above the form (slides down, 200ms). Form fields remain filled so the user can correct and retry. Button returns to "kubectl apply" idle state.

### Homelab Scale Interaction

1. **Unselected state:** All 5 emoji buttons render at equal size (44x44px) with border token border. No description visible below the track.
2. **Tap to select:** The tapped button gets a 2px accent-colored ring (`box-shadow: 0 0 0 2px accent`). The description text fades in below the scale (150ms opacity transition). Previously selected button loses its ring.
3. **Re-tap different emoji:** Ring moves, description updates. Only one selection at a time.

### Country Select Interaction

1. **Default state:** Shows placeholder text "Select country..." in text-secondary color.
2. **Tap to open:** Uses native `<select>` element to leverage the OS-native picker on mobile (iOS action sheet, Android bottom sheet). No custom dropdown — native pickers are faster and more accessible on mobile.
3. **European countries appear first** in the option list (grouped visually with an optgroup label "Europe"), followed by all other countries alphabetically (optgroup label "Other countries").

### Error States

| Error Condition | Trigger | User-Facing Message |
|----------------|---------|---------------------|
| Empty name | Submit with empty name field | "Enter your name to join the cluster" |
| Name too long | Name exceeds 30 characters | "Name must be 30 characters or fewer" |
| No country selected | Submit without country | "Select where you're from" |
| No homelab level | Submit without homelab selection | "Pick your homelab level" |
| Profanity detected | Server returns 422 | "That name wasn't accepted. Try a different one." |
| Network error | Fetch fails (timeout, offline) | "Couldn't reach the server. Check your connection and try again." |
| Server error | 500 response | "Something went wrong. Try again in a moment." |
| Submissions closed | Server returns 403 | Redirect to visualization page (per CONTEXT.md decision) |

Field-level errors appear directly below the offending input in destructive color at 14px. Network/server errors appear as a banner above the form.

### Loading Performance Contract (INFR-04)

- Total frontend bundle (JS + CSS): under 50KB gzipped
- No web fonts loaded (system font stack)
- No images loaded (emojis are Unicode, not image assets)
- Critical render path: single HTML file, single JS bundle, single CSS file
- Target: first meaningful paint under 1.5 seconds on 4G (1.6 Mbps)

## Animations

| Animation | Property | Duration | Easing | Trigger |
|-----------|----------|----------|--------|---------|
| Form to confirmation | opacity | 200ms | ease-out | Successful submission response |
| Error banner enter | transform (translateY) + opacity | 200ms | ease-out | Error received |
| Error banner exit | opacity | 150ms | ease-in | 5 seconds elapsed or user interacts |
| Homelab description | opacity | 150ms | ease-in-out | Emoji button tapped |
| Button loading spinner | transform (rotate) | 800ms | linear, infinite | Submit button in loading state |

No spring physics. No JavaScript animation libraries. CSS transitions and keyframes only. This keeps the bundle under 50KB.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | All interactive elements: minimum 44x44px (WCAG 2.5.5) |
| Focus indicators | 2px accent ring on all focusable elements (visible on dark background) |
| Color contrast | Text primary on dominant: #F8FAFC on #0F172A = 15.4:1 (AAA). Text secondary on dominant: #94A3B8 on #0F172A = 5.6:1 (AA). |
| Screen reader | Homelab emoji buttons have `aria-label` with their description text |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all transitions, shows states immediately |
| Form errors | Errors linked to inputs via `aria-describedby`. Error banner uses `role="alert"` |
| Keyboard nav | Tab order follows visual order: name -> country -> homelab 1-5 -> submit |
