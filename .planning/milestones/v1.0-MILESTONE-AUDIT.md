## Unsatisfied Requirements Detail

### SUBM-06: "Attendee sees a 'You're now a pod in the cluster!' confirmation after submitting"

**Root cause:** Phase 3 (Admin Panel) rewrote `app.tsx` to add `/admin` routing and a top-level `appView` signal. The original Phase 1 flow used `formState === 'success'` to render `Confirmation.tsx`. Phase 3 replaced this with `appView === 'viz'` → `VizPage`, making `Confirmation.tsx` dead code.

**Evidence:**
- `web/src/components/SubmissionForm.tsx:64` — sets `appView.value = 'viz'` on success
- `web/src/app.tsx:39` — `if (appView.value === 'viz') return <VizPage />`
- `web/src/components/Confirmation.tsx` — exists but is never imported by `app.tsx`

**Severity assessment:** The attendee still gets clear feedback — they see their pod appear in the live visualization, which is arguably a more impressive confirmation than a static screen. The Confirmation component's CTA said "Look up at the screen to find yourself!" — the current flow puts them ON the screen. This is a technical regression but potentially a UX improvement.

**Options:**
1. **Accept as-is** — update SUBM-06 wording to match actual behavior ("Attendee sees themselves appear in the visualization")
2. **Restore confirmation** — show Confirmation screen briefly (2-3s), then auto-transition to viz
3. **Delete dead code** — remove `Confirmation.tsx` and update requirement
